## Item 1 — ESLint `max-lines` at `warn` + lint-staged ratchet

**Goal:** stop *new* bloat immediately; grandfather existing files. Nothing breaks on day one (`warn`, not `error`).

### 1a. `scheduler-app/eslint.config.mjs`
Add one line to the existing `commonRules` object (it already flows into both the typed and untyped blocks):

```js
const commonRules = {
  // ...existing...
  "max-lines": ["warn", { max: 500, skipBlankLines: true, skipComments: true }],
};
```

Then append two scoped blocks to the exported array (after the existing blocks):

```js
  // Long-function rule — scoped to logic modules, NOT UI/route/tests
  {
    files: ["src/lib/**/*.ts", "app/api/**/*.ts"],
    rules: {
      "max-lines-per-function": ["warn", { max: 100, skipBlankLines: true, skipComments: true, IIFEs: true }],
    },
  },
  // Exempt legitimately-long / generated / data files
  {
    files: [
      "src/lib/database.types.ts",            // generated by `supabase gen types`
      "scripts/catalog/categories/**/*.ts",   // pure data (one cohesive unit per file) — see Item 2
    ],
    rules: { "max-lines": "off" },
  },
  // Tests + config: function-length rule off (describe blocks are legitimately long)
  {
    files: ["**/*.test.{ts,tsx}", "tests/**", "**/*.config.{ts,mts}"],
    rules: { "max-lines-per-function": "off" },
  },
```

### 1b. `admin-app/eslint.config.mjs`
This one uses `FlatCompat`. Append a rules block after the spread:

```js
const eslintConfig = [
  ...compat.extends("next/core-web-vitals", "next/typescript"),
  { ignores: [".next/**", "node_modules/**"] },
  { rules: { "max-lines": ["warn", { max: 500, skipBlankLines: true, skipComments: true }] } },
];
```

### 1c. Ratchet — block new bloat on changed files only
Repo root already has husky 9 + `prepare: husky`. Add to **root `package.json`**:

```jsonc
"lint-staged": {
  "scheduler-app/**/*.{ts,tsx}": "eslint --max-warnings 0",
  "admin-app/**/*.{ts,tsx}": "eslint --max-warnings 0"
}
```

`.husky/pre-commit` (append): `npx lint-staged`

> ⚠️ **Pre-flight before enabling `--max-warnings 0`:** run `cd scheduler-app && npm run lint` and count the *existing* warning backlog (the config already carries deferred `react-hooks/refs` + `set-state-in-effect` warnings). `--max-warnings 0` blocks a commit on **any** warning on a touched file, not just size ones. If the backlog is non-trivial, either (a) clear it first, or (b) scope the gate to size rules only. **Decision point — confirm with Chris.**

## Item 3 — Extract `_shared/resend-client.ts` (3 sites now, 2 deferred)

### New transport (`supabase/functions/_shared/resend-client.ts`, ~50 L)

```ts
export interface SendEmailArgs {
  from: string;
  to: string | string[];
  subject: string;
  html: string;
  replyTo?: string;
}
export interface SendEmailResult { ok: boolean; status: number; id?: string; error?: string; }

/** Sends one email via Resend. Reads RESEND_API_KEY at call time. Never throws — returns a result. */
export async function sendResendEmail(args: SendEmailArgs): Promise<SendEmailResult> {
  const key = Deno.env.get("RESEND_API_KEY") ?? "";
  if (!key) return { ok: false, status: 0, error: "RESEND_API_KEY not configured" };
  try {
    const res = await fetch("https://api.resend.com/emails", {
      method: "POST",
      headers: { Authorization: `Bearer ${key}`, "Content-Type": "application/json" },
      body: JSON.stringify({
        from: args.from,
        to: Array.isArray(args.to) ? args.to : [args.to],
        subject: args.subject,
        html: args.html,
        ...(args.replyTo ? { reply_to: args.replyTo } : {}),
      }),
    });
    const body = await res.json().catch(() => ({}));
    return res.ok
      ? { ok: true, status: res.status, id: body?.id }
      : { ok: false, status: res.status, error: body?.message ?? `Resend ${res.status}` };
  } catch (e) {
    return { ok: false, status: 0, error: e instanceof Error ? e.message : String(e) };
  }
}
```
