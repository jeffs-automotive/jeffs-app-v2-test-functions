## ⚠️ Diagnosis correction (the confirmation bug)

An earlier hand diagnosis blamed the `KeytagsTabs` **controlled-value + `window.history.replaceState`
desync** for the broken force-assign confirmation. The adversarial verifier **refuted that as the
reachable trigger**: the snap only fires when `defaultValue` actually *changes*, and no navigation in the
tree crosses tabs into a mounted dialog (the two soft `router.push` filters target their own tab; the
board deep-links are plain `<a href>` full reloads). The controlled-tab pattern is real latent fragility,
but **the active cause is the per-tab `<Suspense>` remount** (see #1). Lesson: the first instinct named
the right *family* (form unmount wiping `useActionState`) but the wrong *trigger*; the adversarial pass
caught it.

## BLOCKER

### B1 — Force-assign Pattern-A confirmation is broken in production (Suspense remount)
`admin-app/app/keytags/page.tsx:82-86` · `AssignKeytagForm.tsx:22-29,160-171` · `LiveBoardTab`/`load-board-state.ts:52-56`

`force_assign`'s **first** call is a Server Action that issues the confirmation token + returns
`needs_confirmation`. On the `force-dynamic` `/keytags` route, **every Server Action re-renders the whole
route tree via Flight regardless of `revalidatePath`** — this is runtime-log-proven in the project's own
[`board-spin-REAL-rootcause-2026-06-25.md`](board-spin-REAL-rootcause-2026-06-25.md) (Defect ①), which
**disproves** the "no `revalidatePath` ⇒ no re-render" objection. `cd91ecf` then wrapped each tab in
`<Suspense>`, and `LiveBoardTab.loadBoardState` makes two awaited, **uncached** orchestrator round-trips —
so on the action re-render the Live `<Suspense>` boundary falls back and **unmounts `AssignKeytagForm`**,
resetting its `useActionState` to idle and destroying the `needs_confirmation` state + token **before the
operator can click Confirm**. Token orphaned, no write, no error, one fast 200.

**Fix:** lift `AssignKeytagForm` (force-assign) **out of the suspending `LiveBoardTab` subtree** (render it
from cached/streamed state, or as its own non-suspending island), **or** give `loadBoardState` a
cached/transition-isolated read so the boundary never falls back on an action re-render. Add a **Playwright
E2E** (real browser, not RTL): force-assign → Confirm → assert the second token POST fires and
`consumed_at` is set. Also fix the independent contributor **#14** (clock-skew-disabled Confirm).

## HIGH

### H1 (#2) — Nightly reconcile writes `p_source="reconcile"`, which the CHECK rejects → audit rows silently lost
`keytag-bulk-reconcile/reconcile.ts:399,465,500` · `reverse-reconcile.ts:167,225`

All five post-mutation `log_keytag_audit` calls pass `p_source:"reconcile"`, which the live CHECK never
permitted → each `INSERT` raises `check_violation (23514)`. The keytag state change already committed in a
prior RPC; the audit error is routed to `surfaceRpcError` (logs + Sentry) but **never throws and its
boolean is discarded**, so the mutation returns success while the audit row vanishes. **Live-confirmed:
zero `reconcile` rows.** Weakens L1 prior-history detection. **Fix:** change the 5 sites to `"cron"` (the
documented value); gate the audit-write result so an enum mismatch can't silently lose rows.

### H2 (#3) — `releaseKeytagFromRo` swallows the status-read error → no-confirmation A/R release (L2 bypass, L4 neutralized)
`_shared/tools/keytag-management.ts:434-462,508`

`dbTagStatus` (the A/R gate input) comes from a SELECT that destructures away `error`. On any read failure
the code falls to the Tekmetric else-branch with `dbTagStatus=null` → `requiresConfirmation=false` → the
two-step block is skipped and `release_keytag_for_ro` runs. That RPC unconditionally sets the
`ar_mutation_allowed` GUC, so **L4 permits the `posted_ar→available` transition with no token**. An
A/R-locked tag is freed with no confirmation, **fail-open under DB stress**. **Fix:** capture the error and
**fail closed** (unreadable status → require confirmation / abort).

### H3 (#4) — L1 `work_approved` anti-drift gate fails OPEN on a swallowed audit read → double-assigns a second physical tag
`keytag-tekmetric-webhook/index.ts:694-701` (prior-history read), `937-950` (assigned write)

The Rule-B gate reads `keytag_audit_log` prior history but destructures only `data`; on a read error
`priorHistory` is undefined, the `if (priorHistory)` block is skipped, and control falls through to
`assign_next_keytag` + PATCH — assigning a **brand-new tag while the keys still carry the old number** (the
exact drift L1 exists to stop). Compounded by the `assigned` audit write also discarding its error. **Fix:**
capture the read error and **fail closed** (unreadable audit ⇒ treat as "has history" → DRF/manual-review).

### H4 (#5) — Dashboard "ROs without key tags" skip only matches `claude_desktop` → admin-app A/R releases reappear as false rows
`_shared/keytag-dashboard-data.ts:248-252` · re-issue gate `keytag-bulk-reconcile/reconcile.ts:105-116`

`fetchRosWithoutKeytags` skips an unresolved ARN only when the latest `released` row is
`source==='claude_desktop'`. Since the 2026-06-24 provenance split, an admin-app dashboard release is
`source='admin_app'`, so the skip no longer fires — every dashboard-released A/R tag (Carmax-type fleet)
re-surfaces next morning in the 7 AM email + the live board as a false "needs investigation" row. The
reconcile-side re-issue short-circuit has the identical bug. **Board-fix-series regression.** **Fix:** treat
`{claude_desktop, admin_app}` as human releases in **both** sites; add a regression test.

### H5 (#6) — Three keytag edge functions are anon-key-readable/invocable (two read PII, one **writes**)
`config.toml:441-461,492-494` · `tekmetric-list-wip-keytags`, `tekmetric-find-ro-by-keytag`, `keytag-seed-from-tekmetric`

All three are `verify_jwt=true` with **zero in-handler auth** and build a service-role client (RLS
bypassed). The public publishable/anon key is a signature-valid Supabase JWT, so any browser holding it is
admitted — the project's own B1/B2 comments document this exact hole for the sibling probes (which were
fixed). `list` returns `customer_name` (PII) + ro/customer/vehicle ids for the whole pool; `find`
enumerates any tag→RO/customer/vehicle; **`seed` is a WRITE** that crawls Tekmetric and flips available
tags to assigned/posted_ar from possibly-stale fields. **Fix:** `verify_jwt=false` + `checkSchedulerBearer`
on all three (seed is operator-only); or delete the two diagnostic read probes (the orchestrator dispatches
the functions directly). Add anon-key→401 tests.

  **Fix:** use null/null; make `writeAuditLog` `Sentry.captureException` on failure.
- **M3 (#9) — Silent-failure cluster** on webhook write/outcome paths: `markProcessed` UPDATE, several `log_keytag_audit`,

  reconcile racing the same RO both issue a review + duplicate email. `manual-review.ts:162-210`. **Fix:** partial UNIQUE on

- **M5 (#11) — PAF/DRF resolution falsely tells the advisor "a fresh PAF review has been issued"** on a retry-PATCH

- **M6 (#12) — orchestrator-mcp catch blocks never `Sentry.captureException`** → every keytag/scheduler tool failure is

- **M7 (#13) — OAuth refresh-token reuse detection absent** — a stolen, rotated token leaves the thief's chain valid while
  only the victim is locked out; `parent_token_hash` is written but never read. `mcp-auth/index.ts:592-680`.

## Cross-cutting themes

1. **Board-fix-series regression cluster** — the 06-24/25 spin fixes introduced/re-opened ≥3 live defects (B1 confirmation
   remount, H4 dashboard skip, the controlled-tab/Suspense fragility) by changing re-render/mount behavior without
   re-checking the Pattern-A token lifecycle.
2. **Audit-provenance split incomplete** — the `admin_app` source work threaded source only into assign/release; every other
   path loses it (H1 reconcile invalid value, M1 mislabel, H4 skip) — one half-finished refactor radiates four findings.
3. **Silent failure on defense-relevant reads/writes** — swallowed `error` doesn't just lose logs, it **weakens the four-layer
   defense** (H2 release bypass, H3 double-assign).
4. **`verify_jwt=true` ≠ authorization** — the anon key is a signature-valid JWT; three keytag fns lack any in-handler check.
5. **CHECK-constraint mismatches silently drop audit rows** — `source='reconcile'` (H1) and `tag_number=0` (M2), both swallowed.
6. **Defense-in-depth fragility (latent)** — RLS-zero-policy + standing anon DML grants.
